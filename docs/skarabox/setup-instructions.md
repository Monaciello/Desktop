# Skarabox Host Setup

Guide for adding a skarabox-managed server host.
Reference: [nix-starter-configs-skarabox](https://github.com/ibizaman/nix-starter-configs-skarabox)

**Note:** Alice is a desktop and stays as a regular `nixosConfigurations.alice`.
Only new server hosts go under `skarabox.hosts`.

## Prerequisites

- flake-parts wrapping outputs (done)
- `inputs.skarabox.flakeModules.default` imported (done)
- `nixos-generators`, `nixos-anywhere` inputs uncommented (needed for beacon/install)
- sops-nix configured (see `sops-setup.md`)
- Personal age key at `~/.config/sops/age/keys.txt`

## Quick Start: Add a New Host

The fastest path uses skarabox's built-in scaffolding:

```bash
# 1. Generate host directory with keys and config template
nix run .#gen-new-host <hostname>

# 2. Add host key to .sops.yaml
nix run .#add-sops-cfg -- -o .sops.yaml <hostname> $(ssh-to-age -i ./<hostname>/host_key.pub)

# 3. Edit configuration.nix — set hostname, username, disk layout, IP
$EDITOR <hostname>/configuration.nix

# 4. Set IP and system in flake.nix skarabox.hosts.<hostname>

# 5. Generate known_hosts
nix run .#<hostname>-gen-knownhosts-file

# 6. Git add all new files
git add <hostname>/ .sops.yaml
nix flake check
```

## Host Directory Structure

Each skarabox host needs (auto-generated by `gen-new-host`):

```
<hostname>/
├── configuration.nix  # NixOS module (NOT encrypted)
├── secrets.yaml       # SOPS-encrypted secrets
├── facter.json        # Hardware config (generated after first boot on beacon)
├── host_key           # SSH host private key (NOT committed — .gitignore)
├── host_key.pub       # SSH host public key
├── ssh                # User SSH private key (NOT committed — .gitignore)
├── ssh.pub            # User SSH public key
├── known_hosts        # SSH known hosts (generated)
└── system             # System type (x86_64-linux)
```

**Important:** Private keys (`host_key`, `ssh`) must NOT be committed.
They are strings (not paths) in flake.nix to stay out of the nix store.

## Flake Configuration

```nix
skarabox.hosts = {
  myserver = {
    system = "x86_64-linux";
    hostKeyPub = ./myserver/host_key.pub;
    ip = "192.168.1.30";
    sshPort = 2222;
    sshBootPort = 2223;
    knownHosts = ./myserver/known_hosts;

    modules = [
      inputs.selfhostblocks.nixosModules.default
      inputs.sops-nix.nixosModules.default
      ./myserver/configuration.nix
    ];
  };
};
```

## Host Configuration Template

```nix
# <hostname>/configuration.nix
{ config, ... }:
{
  skarabox = {
    hostname = "<hostname>";
    username = "sasha";
    hashedPasswordFile = config.sops.secrets."<hostname>/user/hashedPassword".path;
    facter-config = ./facter.json;
    sshPort = 2222;
    boot.sshPort = 2223;
    sshAuthorizedKey = ./ssh.pub;

    disks.rootPool = {
      disk1 = "/dev/nvme0n1";
      reservation = "500M";
    };
  };

  sops = {
    defaultSopsFile = ./secrets.yaml;
    age.sshKeyPaths = [ "/boot/host_key" ];
    secrets."<hostname>/user/hashedPassword".neededForUsers = true;
  };
}
```

## Installation Flow

### Option A: VM (testing)

```bash
nix run .#<hostname>-beacon-vm &
nix run .#<hostname>-get-facter > <hostname>/facter.json
git add <hostname>/facter.json
nix run .#<hostname>-install-on-beacon
```

### Option B: On-premise server

```bash
# Build beacon ISO
nix build .#<hostname>-beacon
# Burn to USB
nix run .#beacon-usbimager
# Boot server on USB, then:
nix run .#<hostname>-get-facter > <hostname>/facter.json
git add <hostname>/facter.json
nix run .#<hostname>-install-on-beacon
```

## Post-Install Operations

```bash
nix run .#<hostname>-unlock    # Decrypt root partition after reboot
nix run .#<hostname>-ssh       # SSH to host
nix run .#sops <hostname>/secrets.yaml  # Edit secrets
nix run .#gen-new-host <name>  # Scaffold another host
```

## Troubleshooting

### `nix shell nixpkgs#<pkg>` fails with "does not provide attribute"

Your nix registry may point `nixpkgs` to your flake instead of upstream.
Use `nix develop` to get tools from your dev shell, or use full flake ref:
```bash
nix shell github:nixos/nixpkgs/nixos-25.11#ssh-to-age
```

### `sops metadata not found`

The secrets file exists as plaintext. Delete it and let sops create it fresh:
```bash
rm <hostname>/secrets.yaml
sops <hostname>/secrets.yaml
```

### `nix flake check` can't find a file

Nix flakes only see git-tracked files. Stage new files first:
```bash
git add <hostname>/secrets.yaml <hostname>/facter.json
nix flake check
```

### skarabox CLI tools can't find sops key

Skarabox defaults to `sops.key` at repo root. Either:
- Symlink: `ln -s ~/.config/sops/age/keys.txt sops.key`
- Or set in flake: `skarabox.sopsKeyPath = "~/.config/sops/age/keys.txt";`
  (note: `~` may not expand — use absolute path or symlink)
